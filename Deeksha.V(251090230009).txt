EXPERIMENT 1 – OBJECT DETECTION USING IR SENSOR

AIM

To interface an IR (Infrared) sensor with Arduino and detect the presence of an object using digital output.

THEORY

An IR sensor works on the principle of infrared light reflection. It consists of an IR transmitter (IR

LED) and an IR receiver (photodiode or phototransistor).

• The IR transmitter emits infrared rays.

• When an object comes in front of the sensor, the rays are reflected back.

• The IR receiver detects the reflected rays and produces a digital output.

Most IR sensor modules provide digital output:

• LOW → Object detected

• HIGH → No object detected

IR sensors are widely used in obstacle detection, line-following robots, proximity sensing, and automation systems.

COMPONENTS REQUIRED:

• Arduino Uno

• IR Sensor Module

• Connecting Wires

CIRCUIT DIAGRAM



CODE

int irPin = 2; // IR sensor output pin int ledPin = 13; // LED pin



void setup() {

pinMode(irPin, INPUT);

pinMode(ledPin, OUTPUT);

Serial.begin(9600);

}



void loop() {

int irState = digitalRead(irPin);



if (irState == LOW) { // Object detected digitalWrite(ledPin, HIGH); // LED ON

Serial.println("Object Detected");

} else { // No object digitalWrite(ledPin, LOW); // LED OFF

Serial.println("No Object");

}



delay(500);

}



EXPERIMENT 2– DISTANCE MEASUREMENT USING ULTRASONIC SENSOR

AIM

To interface an ultrasonic sensor with Arduino and measure the distance of an object using ultrasonic sound waves.

THEORY

An ultrasonic sensor measures distance using ultrasonic sound waves (frequency > 20 kHz). It works on the time-of-flight principle.





The sensor has two main pins:

• TRIG (Trigger): Sends ultrasonic pulses

• ECHO (Echo): Receives the reflected pulse Working principle:

1. Arduino sends a short HIGH pulse to the TRIG pin.

2. The ultrasonic sensor emits sound waves.

3. These waves hit an object and reflect back.

4. The ECHO pin stays HIGH for the time taken by the wave to return.

5. Distance is calculated using the formula: Time × Speed of Sound

Speed of sound ≈ 0.034 cm/µs

Ultrasonic sensors are widely used in obstacle detection, distance measurement, parking systems, and robotics.

EXPERIMENT 03– TOGGLE LED USING PUSH BUTTON

AIM

To interface a push button with Arduino and control an LED in toggle mode using a switch.

THEORY

A push button is a digital input device that provides HIGH or LOW logic levels. In Arduino, the internal pull-up resistor can be enabled using INPUT_PULLUP, which keeps the input pin HIGH

when the switch is not pressed.

When the button is pressed, the input pin goes LOW. By detecting the transition from HIGH to LOW, the LED state can be toggled. A small debounce delay is added to avoid multiple toggles due to mechanical bouncing of the switch.

This method allows one button to turn the LED ON and OFF alternately.

COMPONENTS REQUIRED:

• Arduino Uno

• Push Button

• Connecting Wires

CIRCUIT DIAGRAM





CODE

int switchPin = 2;

int ledPin = 13;



bool ledState = LOW;

bool lastSwitchState = HIGH;



void setup() {

pinMode(switchPin, INPUT_PULLUP);

pinMode(ledPin, OUTPUT);

}



void loop() {

bool currentSwitchState = digitalRead(switchPin);



// Detect button press (HIGH to LOW)

if (lastSwitchState == HIGH && currentSwitchState == LOW) {

ledState = !ledState; // toggle LED state digitalWrite(ledPin, ledState);

delay(200); // debounce delay

}



lastSwitchState = currentSwitchState;

}

Interfacing MPR121 Capacitive Touch Sensor with Arduino Using I2C Protocol





AIM: To interface a capacitive touch sensor (MPR121) with Arduino using the I2C

communication protocol and observe touch detection through serial monitoring.

Components Required:

• Arduino UNO

• MPR121 Capacitive Touch Sensor Module

Circuit Diagram:





Theory:

The MPR121 capacitive touch sensor provided in the IoT Trainer Kit is an I2C-based touch controller capable of detecting up to 12 touch electrodes by sensing changes in capacitance when a finger touches the sensor pads. It communicates with the Arduino Uno through the I2C protocol using SDA and SCL lines, enabling reliable and fast data exchange. The sensor continuously monitors electrode capacitance, applies digital filtering, and compares the values with preset touch and release thresholds to accurately detect touch events. When a touch is detected, the Arduino reads the sensor registers via I2C and displays the touched electrode information through serial monitoring.





Code:

#include <Wire.h>

#define MPR121_ADDR 0x5A

uint16_t last=0;

void w(byte r, byte v){

Wire.beginTransmission(MPR121_ADDR);

Wire.write(r); Wire.write(v);

Wire.endTransmission();

}

uint16_t readT(){

Wire.beginTransmission(MPR121_ADDR);

Wire.write(0x00);

Wire.endTransmission(false);

Wire.requestFrom(MPR121_ADDR,2);

return Wire.read() | (Wire.read()<<8);}

void setup(){

Serial.begin(9600);

Wire.begin();

w(0x5E,0x00); // stop

// FULL FILTER CONFIG (important!!)

w(0x2B,0x01); w(0x2C,0x01);

w(0x2D,0x00); w(0x2E,0x00);

w(0x2F,0x01); w(0x30,0x01);

w(0x31,0xFF); w(0x32,0x02);

w(0x33,0x01); w(0x34,0x01);

w(0x35,0xFF); w(0x36,0x02);

// STRONG thresholds

for(int i=0;i<12;i++){

w(0x41+i*2,30);

w(0x42+i*2,15); }

w(0x5E,0x8F); // start

Serial.println("READY"); }

void loop(){

uint16_t now=readT();

uint16_t press=now & ~last;

for(int i=0;i<12;i++)

if(press&(1<<i)){

Serial.print("Touched electrode "); Serial.println(i);

}

last=now;

delay(60);



}

Interfacing Arduino UNO with LDR and 16x2 LCD





AIM:

To interface an LDR sensor with Arduino uno and display the status on LCD

Components Required:



• Arduino UNO

• LDR

• LCD



Circuit Diagram:





Theory:

A Light Dependent Resistor (LDR), also known as a photoresistor, is a passive electronic component whose resistance varies according to the intensity of incident light. It works on the principle of photoconductivity.

When light falls on the LDR surface, photons transfer energy to the semiconductor material (usually cadmium sulfide – CdS), generating free charge carriers. As the number of charge carriers increases, the resistance of the LDR decreases.

• High light intensity → Low resistance

• Low light intensity (darkness) → High resistance Typically:

• In bright light → resistance may drop to a few hundred ohms

• In darkness → resistance may increase to several megaohms





Since Arduino cannot directly measure resistance, the LDR is connected in a voltage divider circuit with a fixed resistor. The varying resistance produces a corresponding change in output voltage, which is read using the Arduino’s analog input (ADC).

The Arduino UNO contains a 10-bit ADC, which converts analog voltage (0–5V) into digital values between 0 and 1023. Based on a predefined threshold value, the system determines whether the environment is bright or dark and displays the result on the LCD.

Code:

#include <Wire.h>

#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27, 16, 2);

int ldrPin = A0;

int threshold = 500; // Adjust based on testing void setup() {

Serial.begin(9600);



lcd.init();

lcd.backlight();

}

void loop() {

int ldrValue = analogRead(ldrPin);

Serial.println(ldrValue);

lcd.clear();

if (ldrValue > threshold) {

lcd.print("Light Detected");

} else {

lcd.print("Light Not");

lcd.setCursor(0,1);

lcd.print("Detected");

}

delay(500);

}





Interfacing Arduino UNO with Touch Sensor (MPR121) and 16x2 LCD

AIM: To interface Arduino Uno with MPR121 capacitive touch sensor and display touched electrode number on 16x2 LCD.

Components Required:

• Arduino Uno

• MPR121

• 16x2 LCD

• Connecting wires



Circuit Diagram:





Theory:

The MPR121 capacitive touch sensor is a touch detection controller that operates based on the principle of capacitance variation. It can detect touch without mechanical pressure by sensing changes in the electrical capacitance of its electrodes. When a human finger approaches or touches an electrode, the capacitance increases because the human body acts as a conductive object. The MPR121 continuously monitors this change using its internal charge-discharge circuitry and compares it with predefined threshold values. If the change exceeds the threshold, a touch event is registered. The sensor supports up to 12 touch electrodes and communicates with the Arduino using the I2C protocol (SDA and SCL lines, typically at address 0x5A). In this experiment, when an electrode is touched, the Arduino reads the touch status data from the MPR121 and displays the corresponding electrode number on the LCD.





Code:

#include <Wire.h>

#include <LiquidCrystal_I2C.h>

#define MPR121_ADDR 0x5A

LiquidCrystal_I2C lcd(0x27, 16, 2);

uint16_t last = 0;

// REVERSED mapping

char keys[12] = {

'#','0','*',

'9','8','7',

'6','5','4',

'3','2','1'

};

// Write register

void w(byte r, byte v){

Wire.beginTransmission(MPR121_ADDR);

Wire.write(r);

Wire.write(v);

Wire.endTransmission();

}

// Read touch status

uint16_t readT(){

Wire.beginTransmission(MPR121_ADDR);

Wire.write(0x00);

Wire.endTransmission(false);

Wire.requestFrom(MPR121_ADDR,2);

return Wire.read() | (Wire.read()<<8);

}



void setup(){

Wire.begin();

lcd.init();

lcd.backlight();

lcd.print("Keypad Ready");

// Stop MPR121

w(0x5E,0x00);

// Filter config

w(0x2B,0x01); w(0x2C,0x01);

w(0x2D,0x00); w(0x2E,0x00);

w(0x2F,0x01); w(0x30,0x01);

w(0x31,0xFF); w(0x32,0x02);

w(0x33,0x01); w(0x34,0x01);

w(0x35,0xFF); w(0x36,0x02);

// Strong thresholds

for(int i=0;i<12;i++){

w(0x41+i*2,30);

w(0x42+i*2,15);

}

// Start

w(0x5E,0x8F);

delay(1500);

lcd.clear();

}

void loop(){

uint16_t now = readT();

uint16_t press = now & ~last;

for(int i = 0; i < 12; i++){

if(press & (1 << i)){

lcd.clear();

lcd.setCursor(0,0);

lcd.print("Pressed:");

lcd.setCursor(0,1);

lcd.print(keys[i]);

}

}

last = now;

delay(60);

}

RASPBERRY PI – ULTRASONIC SENSOR

AIM: To interface an ultrasonic sensor with Raspberry Pi and measure the distance of an object using ultrasonic sound waves.

COMPONENTS REQUIRED:

• Raspberry Pi

• Ultrasonic sensor

CIRCUIT DIAGRAM:





CODE:

import RPi.GPIO as GPIO

import time

# Set GPIO mode



GPIO.setmode(GPIO.BCM)

# Define GPIO pins

GPIO_TRIGGER = 4 # Trigger pin (GPIO4 - Pin 7) GPIO_ECHO = 17 # Echo pin (GPIO17 - Pin 11)

# Setup GPIO direction

GPIO.setup(GPIO_TRIGGER, GPIO.OUT)

GPIO.setup(GPIO_ECHO, GPIO.IN)

# Function to calculate distance

def distance():

# Set Trigger HIGH

GPIO.output(GPIO_TRIGGER, True)

# Set Trigger LOW after 10 microseconds

time.sleep(0.00001)

GPIO.output(GPIO_TRIGGER, False)

start_time = time.time()

stop_time = time.time()

# Save start time

while GPIO.input(GPIO_ECHO) == 0:

start_time = time.time()

# Save time of arrival

while GPIO.input(GPIO_ECHO) == 1:

stop_time = time.time()

# Time difference

time_elapsed = stop_time - start_time

# Distance calculation (cm)

distance_cm = (time_elapsed * 34300) / 2

return distance_cm





try:

print("Ultrasonic Sensor Measurement Started") while True:



dist = distance()

print(f"Measured Distance = {dist:.2f} cm") time.sleep(1)

except KeyboardInterrupt:

print("Measurement stopped by User") finally:

GPIO.cleanup()

Interfacing Raspberry Pi with ADC (MCP3208) AIM: To interface an Analog-to-Digital Converter (ADC) with a Raspberry Pi using SPI communication and read analog voltage from an external sensor or potentiometer.

COMPONENTS REQUIRED:

• Raspberry Pi

• ADC IC (MCP3208 – 12-bit)

• Potentiometer / Analog Sensor

CIRCUIT DIAGRAM:



CODE:

import spidev

import time

# SPI setup

spi = spidev.SpiDev()

spi.open(0, 0) # Bus 0, CE0

spi.max_speed_hz = 1000000

channel = 0

def read_adc(channel):

"""Read ADC value from MCP3208 (12-bit)"""

if channel < 0 or channel > 7:

return -1

start_bit = 0x04

sgl_diff = 0x02

command = start_bit | ((sgl_diff | (channel >> 2)) & 0x03) data = [command, (channel & 0x03) << 6, 0x00]

response = spi.xfer2(data)

result = ((response[1] & 0x0F) << 8) | response[2]

return result

def read_voltage(channel, vref=3.3):

adc_value = read_adc(channel)

voltage = (adc_value / 4095) * vref

return voltage

try:

while True:

voltage = read_voltage(channel)

print("ADC Voltage:", round(voltage, 2), "V") time.sleep(0.2)

except KeyboardInterrupt:

print("\nProgram stopped")



finally:

spi.close()
